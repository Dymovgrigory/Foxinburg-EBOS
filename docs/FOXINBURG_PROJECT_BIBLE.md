# 🦊 FOXINBURG EBOS — Project Bible v1.0

## 1. Главная цель

Создать **FOXINBURG EBOS** (Educational Business Operating System) — единую живую цифровую экосистему для образовательного бизнеса мирового уровня.

Это НЕ сайт школы, НЕ просто LMS, НЕ просто CRM. Это операционная система, в которой каждый пользователь может прожить свой рабочий или учебный день, не выходя из системы.

## 2. Что заменяет FOXINBURG EBOS

- ❌ Trello / ClickUp / Notion
- ❌ Google Drive / Яндекс Диск как отдельный инструмент
- ❌ AmoCRM / Bitrix24 / BigBen CRM
- ❌ Отдельная LMS
- ❌ Отдельная ERP / HRM
- ❌ Отдельный корпоративный университет
- ❌ Excel-отчёты и Google-таблицы

## 3. Четыре направления системы

### 3.1 BUSINESS (управление бизнесом)
**Роли:** OWNER, SUPER_ADMIN, ADMIN, MANAGER

- **CRM:** лиды, воронки, сделки, продажи, заявки, телефония, история взаимодействий
- **ERP:** финансы, бюджет, расходы, доходы, кассы, платежи, филиалы
- **HR:** сотрудники, должности, графики, отпуска, KPI, задачи
- **Analytics:** отчёты, аналитика, прогнозы, KPI

### 3.2 EDUCATION (образовательная платформа)
**Роли:** METHODIST, TEACHER, ADMIN, сотрудники, внешние клиенты

- **Academy:** курсы с последовательностью: Лекция → Видео → Практика → Тест → Домашнее задание → Проверка методистом → Допуск → Следующий модуль
- **Поддержка форматов:** PDF, DOCX, презентации, видео, задания, тесты, сертификаты
- **Teacher Academy:** обучение педагогов и администраторов школы

### 3.3 STUDENT WORLD (игровая вселенная)
**Роль:** STUDENT

- **Миры:**
  - 🌲 Лес Знакомств (A1)
  - 🏘 Город Общения (A2)
  - 🏰 Королевство Уверенности (B1)
  - 🏛 Академия Мастеров (B2)
  - 🌍 Империя Свободного Английского (C1)
  - 👑 Лига Экспертов (C2)
- **Механики:** NPC, Boss Battles, AI Roleplay, Collectibles, Achievements, Stickers, Coins, Daily Streak, Adaptive Difficulty, Mastery System, World Map, 1000+ заданий

### 3.4 AI ECOSYSTEM
Единый искусственный интеллект, работающий во всей системе:
- AI Teacher
- AI Administrator
- AI Methodologist
- AI Sales Manager
- AI Student Tutor
- AI Owner Assistant

## 4. Роли системы

Всего 9 ролей. Каждая должна быть доведена до 100% готовности:

1. **OWNER** — владелец бизнеса
2. **SUPER_ADMIN** — технический контроль
3. **ADMIN** — управление школой
4. **METHODIST** — методология, проверка ДЗ, программы
5. **TEACHER** — педагог
6. **MANAGER** — продажи, CRM
7. **STUDENT** — ученик
8. **PARENT** — родитель
9. **GUEST** — неавторизованный пользователь

## 5. Требование: ROLE 100% COMPLETION

Каждая роль должна иметь:
1. Полностью рабочий Dashboard (живой, не статичный)
2. Рабочий день пользователя (сценарий от входа до выхода)
3. Все кнопки работают (запрещены: "В разработке", "Скоро", "Coming Soon", заглушки, пустые страницы)
4. Все данные сохраняются в backend → database → API → frontend (запрещён localStorage как источник истины)

## 6. Конструктор системы

OWNER и SUPER_ADMIN должны управлять системой без программиста:
- Создавать сотрудников, филиалы, курсы, программы, материалы
- Создавать группы, учеников, педагогов, методистов
- Создавать задачи, события, шаблоны, уведомления
- Управлять ролями, правами доступа
- Создавать отчёты, KPI, автоматизации, маршруты, AI сценарии

## 7. Отчёты

Автоматическое создание отчётов в форматах PDF, XLSX, CSV:
- Финансовые (P&L, доходы, расходы, прибыль)
- Образовательные (посещаемость, успеваемость, завершение курсов)
- HR (нагрузка, KPI, эффективность)
- CRM (конверсия, продажи, лиды)
- Академия (прогресс обучения)
- Игровой мир (XP, уровни, достижения)

## 8. Философия разработки

- Мы НЕ создаём сайт школы, CRM, LMS, Duolingo или BigBen CRM
- Мы создаём единую операционную систему FOXINBURG EBOS
- Каждый раздел взаимосвязан
- Нет мёртвых кнопок, дублей, заглушек
- 100% рабочий функционал
- Масштабируемая, отказоустойчивая архитектура

## 9. Интеграция бизнес-логики BigBen

**Абсолютное правило:**
- BigBen используется ТОЛЬКО как источник лучших бизнес-процессов
- Foxinburg EBOS остаётся главной системой
- НЕ создавать новую архитектуру, НЕ менять дизайн, роли, модули
- Внедрять логику BigBen внутрь существующей системы

## 10. Технические принципы

- **Архитектура:** Единая, масштабируемая, не дублируется
- **База данных:** PostgreSQL, единая схема
- **Backend:** FastAPI (Python), чёткая модульная структура
- **Frontend:** React + Vite + TypeScript + Tailwind CSS
- **Инфраструктура:** Российская (Yandex Cloud)
- **Интеграции:** Яндекс Диск, Яндекс GPT, Telegram, Zoom, Wazzup, VK, SMS, Email
- **Стили:** CSS/Tailwind создаются один раз и не меняются
