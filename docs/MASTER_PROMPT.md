**FOXINBURG EBOS** — это НЕ сайт школы, НЕ просто LMS, НЕ просто CRM. Это операционная система образовательного бизнеса мирового уровня, которая объединяет:

- Систему аналитики и отчётности
- Систему управления компанией

### 2.1 Четыре направления

1. **BUSINESS** — OWNER, SUPER_ADMIN, ADMIN, MANAGER
   - CRM, ERP, HR, Analytics
2. **EDUCATION** — METHODIST, TEACHER, ADMIN

3. **STUDENT WORLD** — STUDENT

### 2.2 Роли системы (9 штук)

`OWNER`, `SUPER_ADMIN`, `ADMIN`, `METHODIST`, `TEACHER`, `MANAGER`, `STUDENT`, `PARENT`, `GUEST`

### 5.3 База данных

**PostgreSQL** — единственный источник истины.

Основные сущности:
- `users` — пользователи системы
- `roles` / `permissions` — роли и права
- `organizations` / `branches` — организации и филиалы
- `courses` / `modules` / `lessons` — курсы, модули, уроки
- `enrollments` — зачисления
- `groups` — учебные группы
- `homeworks` / `homework_reviews` — ДЗ и проверки
- `leads` / `deals` — лиды и сделки
- `payments` / `transactions` — платежи и транзакции
- `events` / `notifications` — события и уведомления
- `achievements` / `rewards` — достижения и награды

### 5.4 Event-Driven Architecture

Создать движок событий:

```python
class SystemEvent:
    type: str           # LeadCreated, LessonCompleted, PaymentReceived...
    payload: dict
    user_id: int
    created_at: datetime
```

События запускают автоматизации, уведомления, геймификацию.

## 7. ПЕРВООЧЕРЕДНЫЕ МОДУЛИ

### 7.1 Teacher Academy (приоритет #1)

Курсы для обучения педагогов и администраторов школы.

Типы уроков:
1. **Текстовый / файл** — статья, PDF, DOCX
2. **Видео** — видео-урок
3. **Тест** — автоматическая проверка
4. **Домашнее задание** — отправляется методисту на проверку

**Правило блокировки:** пока ДЗ не проверено и не одобрено, переход к следующему уроку/модулю **закрыт**.

Функционал:
- Список курсов
- Карточка курса с прогрессом
- Просмотр урока
- Прохождение теста
- Отправка ДЗ (текст + файлы)
- Проверка ДЗ методистом (оценка, комментарий, статус)
- Допуск к следующему уроку
- Сертификат по завершению

### 7.2 Админ-панель (приоритет #2)

Профессиональная админ-панель для OWNER/SUPER_ADMIN/ADMIN.

Возможности:
- Управление пользователями (CRUD, блокировка, смена ролей)
- Управление филиалами
- Управление курсами и программами
- Управление группами
- Управление справочниками (источники лидов, статусы, уровни и т.д.)
- Настройки системы
- Просмотр логов и аудита

### 7.3 Интеграции (приоритет #3)

- **Яндекс Диск** — хранение файлов, видео, материалов
- **Яндекс GPT** — AI-помощник в системе
